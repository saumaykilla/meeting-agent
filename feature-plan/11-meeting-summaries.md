# 11 — Meeting Summaries

**Depends on:** 09 — LiveKit Agent (generates the summary), 10 — Pinecone RAG (indexing)  
**Blocks:** Nothing  
**Estimated effort:** Small-Medium

---

## Goal

Build the frontend screens for viewing and browsing meeting summaries. Summaries are generated by the Python agent and written to SpacetimeDB. The frontend subscribes to the `meeting_summaries` table and renders them in a clean, document-like view.

---

## Screens Covered

| Screen | Route |
|---|---|
| Summaries List | `/summaries` |
| Summary Detail | `/summaries/[id]` |

---

## Data Flow

```
1. Meeting ends
2. Agent generates summary (Feature 09)
3. Agent writes to SpacetimeDB:
   - meeting_summaries table (structured data)
   - messages table (MeetingThread) as an agent message
4. Frontend subscription fires:
   - /summaries page gets new row
   - /meetings/[id] page shows "View Summary" button
   - Meeting thread in /chat shows CC Assistant message
```

### Keyword Tag Extraction

```typescript
// Extract 3-5 topic tags from key_decisions JSON array
// Show as pill badges: pill, #E8E5DF bg, charcoal text, small font
function extractTags(keyDecisions: string[]): string[] {
  return keyDecisions
    .flatMap(d => d.split(/[,;]/))
    .map(w => w.trim())
    .filter(w => w.length > 3 && w.length < 20)
    .slice(0, 5)
}
```

### Client-side Search

```typescript
function filterSummaries(summaries, searchQuery) {
  if (!searchQuery) return summaries
  const lower = searchQuery.toLowerCase()
  return summaries.filter(s =>
    s.summary_text.toLowerCase().includes(lower) ||
    s.key_decisions.toLowerCase().includes(lower) ||
    meetings[s.meeting_id]?.title.toLowerCase().includes(lower)
  )
}
```

## Components

### `SummaryCard.tsx`

```typescript
// Used on the summaries list page
export function SummaryCard({ summary, meeting, participants }) {
  const decisions = JSON.parse(summary.key_decisions) as string[]
  const tags = extractTags(decisions)
  
  return (
    <Card accentBar="violet">
      <div className="summary-card-header">
        <h3>{meeting.title}</h3>
        <span className="muted">{formatDate(meeting.scheduled_at)} · {duration}m</span>
      </div>
      <AvatarGroup users={participants} max={4} />
      <p className="summary-preview">{truncate(summary.summary_text, 140)}</p>
      <div className="tag-row">
        {tags.map(tag => <Badge key={tag}>{tag}</Badge>)}
      </div>
      <Link href={`/summaries/${summary.id}`}>View Full Summary →</Link>
    </Card>
  )
}
```

## Notes

- `key_decisions` and `action_items` are stored as JSON strings in SpacetimeDB. Always parse with try/catch — the agent may occasionally produce malformed JSON that needs graceful fallback.
- Summary pages are the primary "value delivery" moment for CC. The design must feel polished — this is where users see the ROI of the product.
- The `RecallAlert` card is shown when the meeting's thread contains a recall-type agent message. Parse the meeting thread messages and check for `type === 'recall'` in the content JSON.
